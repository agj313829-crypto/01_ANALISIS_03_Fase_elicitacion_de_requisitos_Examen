¿Qué es la Elicitación de Requisitos?

La elicitación de requisitos es una fase fundamental dentro de la ingeniería de software que consiste en identificar, recopilar y comprender las necesidades, expectativas y restricciones de los usuarios, clientes y demás interesados de un proyecto. Su principal objetivo es obtener información precisa sobre los problemas que se desean resolver y las funcionalidades que el sistema debe ofrecer para satisfacer las necesidades del negocio. Esta etapa sirve como base para el análisis, diseño y desarrollo del software, ya que permite definir claramente qué se espera del sistema antes de comenzar su construcción.

Para llevar a cabo la elicitación de requisitos, se utilizan diversas técnicas como entrevistas, encuestas, observación directa, talleres de trabajo, reuniones con los interesados y creación de prototipos. A través de estas actividades, los analistas recopilan información relevante que posteriormente será documentada y validada. Una elicitación adecuada ayuda a evitar malentendidos entre los usuarios y el equipo de desarrollo, reduce errores durante el proyecto y aumenta las probabilidades de éxito del software al garantizar que el producto final responda realmente a las necesidades de quienes lo utilizarán.

Objetivos

Conocer el dominio del problema

Permite al analista comprender el contexto del negocio y comunicarse de manera efectiva con clientes y usuarios para entender sus necesidades.

Conocer el sistema actual

Implica estudiar el sistema existente, ya sea manual o informatizado, identificando sus fortalezas, debilidades y oportunidades de mejora.

Identificar necesidades y expectativas

Busca descubrir tanto las necesidades explícitas como las implícitas de clientes y usuarios, así como sus expectativas respecto al nuevo sistema que se desarrollará.

Planeación de la Elicitación

La planeación permite definir las actividades, técnicas, participantes y recursos que se utilizarán para recopilar los requisitos del sistema.

  • Identificar fuentes de información.
  • Identificar stakeholders.
  • Crear matriz de interesados.
  • Seleccionar técnicas.
  • Planificar la captura de requisitos.

Fuentes de Requerimientos

Son todos los elementos que aportan información para definir los requisitos del sistema.

👥 Usuarios

Personas que utilizarán el sistema y conocen sus necesidades.

🏢 Clientes

Organizaciones o personas que solicitan el desarrollo.

📄 Documentos

Manuales, formularios y reportes existentes.

💻 Sistemas

Aplicaciones actuales que sirven como referencia.

🎓 Expertos

Personas con amplio conocimiento del negocio.

Stakeholders

Son las personas u organizaciones que tienen interés o influencia en el proyecto de software.
